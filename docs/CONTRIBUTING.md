# Contributing to Tredgate Loan

Thank you for your interest in contributing to Tredgate Loan! This document provides guidelines and information for contributors.

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Install dependencies with `npm install`
4. Create a new branch for your feature or fix

## Development Workflow

### Running the Development Server

```bash
npm run dev
```

### Running Tests

```bash
npm run test
```

### Linting

```bash
npm run lint
```

### Building

```bash
npm run build
```

## Code Style Guidelines

- Use TypeScript for all new code
- Follow existing patterns in the codebase
- Keep functions small and focused
- Add appropriate type definitions in `src/types/`

## Pull Request Process

1. Ensure your code passes linting and tests
2. Update documentation if needed
3. Write a clear description of your changes
4. Request a review from maintainers

## Reporting Issues

When reporting issues, please include:

- A clear description of the problem
- Steps to reproduce
- Expected vs actual behavior
- Browser and environment information (if applicable)

## Questions?

Feel free to open an issue for any questions about contributing.
